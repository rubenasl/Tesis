    Qu es un SGBD?

    Para explicar las funciones de un gestor de base de datos o SGBD, primero comenzaremos explicando de qu se trata.

    Un gestor de base de datos o SGBD es un software constituido por una serie de programas cuya funcionalidad es crear, gestionar y administrar la informacin contenida en una base de datos. Es decir, un SGBD tiene como objetivo servir de interfaz entre los usuarios y las aplicaciones.

    Este software se compone de un lenguaje de manipulacin, de un lenguaje de definicin de datos y de un lenguaje de consulta, de forma que tiene la capacidad de trabajar a distintos niveles pero es invisible para el usuario final.

    Gracias a las funciones de un gestor de base de datos los usuarios pueden crear, actualizar, leer y eliminar la informacin de la empresa o negocio. De esta forma, la calidad de los datos ser ms fiable y estarn organizados de manera ms consciente.

    Qu funcionalidades tiene un gestor de base de datos?

    Ahora que ya sabes qu es un SGBD, pasaremos a explicar las funciones de un sistema de gestin de bases de datos:

    • El gestor de base de datos debe proporcionar a los usuarios la posibilidad de almacenar los datos, acceder a ellos, generar informes y, adems, poder manipularlos. Se trata de la funcin principal de un SGBD.
    • Un SGDB debe definir los datos proporcionados, es decir, que las descripciones de los datos puedan ser almacenados segn su tipo o estructura. A esta funcin se le denomina diccionario de datos o metadatos.
    • La informacin que contiene una base de datos suele ser de carcter sensible o confidencial, por esta razn, una de las funciones de un gestor de base de datos fundamentales es garantizar la seguridad de la informacin almacenada, evitando as ataques que puedan destruir o manipular la informacin, tanto de usuarios autorizados como de usuarios no autorizados.
    • Normalmente los SGDB estn diseados con un complejo sistema de seguridad con los que se pueden otorgar distintos permisos a los usuarios.
    • Adems de garantizar la seguridad, otra de las funcionalidades de los sistemas de gestin de datos es adoptar ciertas medidas para preservar y proteger la integridad de los datos almacenados, garantizando de esta forma su validez.
    • La abstraccin de la informacin es otra de las funciones de un gestor de base de datos. Mediante esta funcionalidad, un SGBD simplifica el sistema de almacenamiento fsico de datos ahorrando de esta forma muchos detalles a los usuarios, independientemente de que se trate de un solo archivo o de varios. Asimismo, se definen varios niveles de abstraccin.
    • Un SGBD tiene la capacidad de modificar el esquema fsico o lgico de una base de datos y sin la necesidad de tener que realizar cambios en las aplicaciones que se sirven de ella. A esta funcin se la conoce como independencia.

    • Para aquellos casos en los que no se puede conseguir la redundancia nula, un sistema de gestin de base de datos llevar un control de la informacin repetitiva, de forma que se actualicen de forma simultanea para evitar confusiones e incongruencias que puedan indicar a error.
    • A colacin con lo mencionado anteriormente, un SGBD contiene distintas funciones que permiten actualizar la base de datos y recuperar aquella informacin que se haya podido perder a partir de copias de seguridad.

    Qu es un SGBD?: tipos de sistemas de gestin de bases de datos

    Saber qu es un SGBD y conocer las funciones de un sistema de datos es fundamental, pero conocer las distintas tipologas te ayudar a saber escoger cul es la que mejor puede encajar con las necesidades de tu empresa o negocio.

    Dependiendo de las caractersticas de un sistema de gestin de base de datos se puede agrupar en una u otra tipologa. Estos son los principales sistemas de gestin de datos que existen:

    Sistemas de bases de datos relacionales

    Una base de datos relacional es un software que permite el acceso y proporcional informacin de datos que estn relacionados entre s.

    En este tipo de bases de datos, cada fila que aparezca en una tabla es una ID nica (clave) y las columnas contienen los atributos de los datos. Para cada registro se asigna un valor de atributo, de esta forma se simplifica la creacin de relaciones entre los puntos de datos.

    Una base de datos en red contiene un gran repertorio de registros relacionados entre s, creando de esta forma una red. Al igual que los sistemas de datos relacionales, cada registro contiene un nico atributo.

    Sistemas de bases de datos jerrquicos

    Como su propio nombre indica, este tipo de bases de datos organiza la informacin en una estructura jerrquica, similar a la forma de un rbol pero al revs. Suelen ser utilizados para gestionar grandes cantidades de datos.

    Sistemas de bases de datos orientados a objetos

    Los sistemas de bases de datos orientados a objetos representan la informacin en forma de objetos, es un sistema utilizado sobre todo en programacin dedicado a objetos.
